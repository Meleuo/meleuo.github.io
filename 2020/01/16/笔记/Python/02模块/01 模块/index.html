模块

为什么要用模块**

有一些复杂的功能普通开发者无法开发完成的，就可以借助别人已经写好的模块来实现。

实现一些Python本身自带的函数或方法无法实现的功能

  1. 操作系统交互
  2. 和时间交互
  3. 取相对应的随机数
  4. 跟网络通信交互

模块的形式：文件夹/py文件/C语言编译好的编译文件

为什么要有模块

  1. 方便开发者分类管理
  2. 可以有效的节省内存
  3. 提供更多的功能

模块别名

导入模块时可以给他设置一个别名的

需要注意的是，这个别名取代的是当前代码内存空间 中指向 模块代码内存地址 的变量名，也就是说，设置了别名的模块原来的指向 模块代码内存地址 的变量名是被别名取缔了也就用不了了

from导入

from module import method 发生了什么

from module import method 和直接import module一样，模块都会被运行一遍加载到那个属于他的内存空间里，

  • 不同的是import module 运行完了模块代码后，当前文件的内存空间记录的是变量 对应的是模块内存地址

  • from module import method 虽然也会开辟一块内存空间记录模块的运行

    不过在 当前文件的内存空间只会记录，指定模块中方法 对应的内存地址

from 命名空间的转换

模块中的方法在任何情况下只能调用到模块自己定义的变量，除非外部修改模块中的变量

同理，在当前代码定义任何变量都不会影响模块中方法的运行，(PS: 这里指的是模块中方法的运行，不是运行模块方法，如果你定义个变量覆盖掉了当前内存空间中 指向模块方法内存地址的变量 这样方法就运行不了)

as 重命名

* 和 __all__

from module import * 就是得是把模块 中所有方法都引用到当前文件里，

__all__就是 *的 生死之敌，且只是*的生死之敌，__all__的作用是定义在模块头部，允许* 可以导入成功的方法

模块相关

###

Python中 py 文件有2种执行方法

  • 以脚本方式执行

    我们常用的pycharm

    和命令行下 python /123.py 都属于脚本方式执行

  • 以模块方式执行

    只有一种方式是以模块方式运行的，就是当py文件被 import导入时，from 也属于import导入。

__name__ & sys.module

  • sys.module 是sys模块的一个方法，sys.module以字典格式存储的当前文件 可以调用的所有模块，字典中的value对应的模块内存地址，key对应的则是模块内存地址的变量名

    并且sys.module 无论在哪里，sys.module["__main__"]对应的都是当前以脚本执行文件的内存地址。

    所以说：

    当一个 py文件以脚本方式执行的时候，sys.module["__main__"]，对于的就是他的内存地址

    当一个 py文件以模块方式运行时，sys.module["__main__"]对应的就是调用他的那个文件的内存地址，而模块自己的内存地址存在sys.module["模块名"]当中

  • __name__ 方法返回的是 当前文件在sys.module 中key的变量名。

把模块当脚本运行

没好说的。直接来条原则

在编写py文件时，所有不在函数和类中封装的内容都应该写在if __name__ == "__main__" 下面

反射本模块中的变量

学习反射的时候我们知道通过 sys.module["__main__"] 可以拿到当前文件的内存地址，从而进行反射，模块被调用时,自己的在sys.module并不是__main__，而是模块自己的文件名，

但是__name__ 方法返回的是 当前文件在sys.module 中key的变量名。

所以说在模块中进行反射，就可以借助__name__这个方法

重新加载模块

没有介绍的价值

pyc编译文件

当python执行import 模块名 时，就会在当前目录下的pycache中生成一个编译好的文件

如果__pycache__不存在，会默认创建的

为什么要编译生成呢，python执行时，其实也是在编译文件，只是编译存放在内存

但是模块呢，在Python认为模块是一个常用又不常修改的py文件，那么他就有一个机制，执行的时候会把编译文件存放在指定的目录中，等下次执行调用模块的时候就可以直接调用这个文件，就不用重新编译模块文件了

这样到以后做比较大的项目的时候，会有效的提升代码的执行顺序

那么如果模块文件被修改了呢，放心，Python会自动的重新生成编译文件覆盖过去

注意：提高的代码效率是指代码中模块的导入效率，并不是代码的其它效率
