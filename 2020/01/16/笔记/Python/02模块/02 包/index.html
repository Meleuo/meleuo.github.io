包

什么是包

集合了一组提供复杂功能的py文件，每个包下一定会有一个__init__.py文件

为什么要用包

当py文件提供的功能很复杂，一个py文件写不下的时候

集合了一组的py的文件，提供了一组复杂功能的文件夹

导入包

当我们想导入包下的某个模块文件时，我们可以直接使用import 导入他。

但是一个包要怎么导入？

如果我们直接导入当前目录下的一个包，那么导入就是这个包的__init__.py文件，是不会报错的，但是__init__.py文件里面默认是什么也没有的，也就是说，虽然我们导入了包，但这个包下的如何模块&方法都是用不了的~

问： 那么如果解决这个问题呢？

答：和__init__.py 进行py交易

绝对路径导入

从包中导入模块，要注意这个包的所在目录，如果判断可以使用sys模块中的path方法(sys.path)

那要怎么写，py执行在那里，就会从那里开始找

相对路径导入

相对路径导入就比较简单，也没推荐使用，因为各自都有缺点

使用相对路径的模块文件不能当成脚本执行，简单来说模块文件使用了相对路径来导入，那这个文件就永远只能当成模块

注意

如果是直接导入一个包，那么相当只是执行了这个包的init文件

并不会帮你把这个包下面的其他包或py文件自动导入

如果想要直接导入包后，包下所有的包以及py文件都能自动导入

那么就要处理编写init文件来实现
