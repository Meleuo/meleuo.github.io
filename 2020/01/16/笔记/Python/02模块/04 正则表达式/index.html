正则表达式

正则表达式是在各个语言中通用的语法，简单说就是使用自定义的规则去匹配出想要的字符串

  • 正则表达式普通的应用场景就是Web开发中的表单验证，在脚本中进行字符串截取。
  • 爬虫领域需要非常精通正则表达式，正则表达式在爬虫领域应用的极为广泛。

正则表达式分为3部分，分别是：字符组，元字符，量词

字符组

字符组 说明
[….] 字符组

每个字符组对应匹配一个字符

注意：在正则表达式中 的a-z，顺序不是常规的abcd…到z，他是按照ASCII码从小到大排列的，所以说[A-z]不仅会匹配所有字母，还会ASCII码上一些奇奇怪怪的东西~这也是为什么[0-9 A-z]能匹配到下划线的原因

元字符

元字符 说明
\d 表示匹配任意数字，等同于[0-9]
\w 匹配数字，字母，下划线，等同于[0-9 A-z]
\s 匹配所有空字符，空格，回车，制表
\D 取反\d
\W 取反\w
\S 取反\s
元字符 说明
\t 匹配制表符
\n 匹配回车
’‘ 匹配空格
元字符 说明
\ 转义符
元字符 说明
^ 匹配字符串的开头
$ 匹配字符串的结尾
元字符 说明
a\ b 匹配a或者b
元字符 说明
[^…] […]取反
元字符 说明
. 匹配除了换行符外的所有字符
() 匹配括号内的表达式，也表示()内的表达式为一个组

量词

量词是在字符组和元字符后使用的。

量词 说明
? 前一个规则匹配0次会1次
+ 前一个规则匹配1次或无数次
* 前一个规则匹配0次或者无数次
量词 说明
{n} 前一个规则匹配次数为 n
{n,} 前一个规则匹配次数最少为 n
{n,m} 前一个规则匹配次数最小为n，最大为m
