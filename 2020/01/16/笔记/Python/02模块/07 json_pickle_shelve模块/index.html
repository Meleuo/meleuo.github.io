序列化模块

json,pickle,shelve 都属于序列化模块

什么是序列化？简单来说就把原本属于：dcit，list，set，tup转换为字符串格式

反序列话：把原本是序列转为的字符串，转会原本的序列数据类型

为什么要用序列化

1：对应文件系统本身而言他可不认识你所谓的各种数据类型，对于他而言所有数据都是字符串，所以数据存文件或者DB里面时必须转换为字符串形式才能存储。

2：数据进行网络传输时，网线上传输的是电流，电压对应二进制，二进制在各种转换出来的结果也是字符串。

所以序列化的目的就是为了让数据能够进行存储，进行网络传输

json 模块

json是编程中最常用的数据格式

  • 支持所有的编程语言

    • 支持的数据类型较少

  • json ≠ 字符串化dict 这里牵扯的比较多，不是本章学习的重点

序列化

json 序列化 说明
json.dumps() 序列化数据
json.dump(objict,fb) open()中使用的，将序列化后的数据传递到文件中

序列化选项

常用的序列化选项

ensure_ascii=Ture(delault)|False

  • Ture 把中文编译成ASCII码
  • False不编译

不常用的序列化选项 没记

反序列化

json 反序列化 说明
json.loads() 反序列化
json.load() open()中使用的，将文件的数据反序列化

Json的限制

json 是一种数据格式(敲黑板)，并且是所有的编程语言都认识的数据格式，json模块是将 Python中的dict类型和 dict长相的 list 转为json

json 虽然和dict长得像但是其区别还是蛮多的

因此json为了兼容所有的编程语言就有诸多的限制

在dict中str是 ‘’或者”“ 都无所谓，但是在json 字符串必须是 使用 ""

json的key不支持数字类型

如果你的dict key为int，使用 .jump 序列化时 就会变成 str，反序列后依旧还是str，value是支持数字类型的

json对元组的支持

tup作为value时使用.dumps序列化后会变成一个列表。

在json中tup不能作为key

pickle 模块

json 兼容所有的开发语言，因此使用json就可以和其他语言交互，但限制也比较多

pickle 模块就是纯粹为了python自己服务

  • pickle 序列化的数据都是bytes类型的，使用pickle序列化的数据只能通过反序列话查看

  • pickle支持几乎支持python中的所有对象，包括变量，类对象等

  • 对于对象的序列化后要进行反序列化，需要这个对象对应的类在内存中
  • dump出来的结果是bytes类型的数据，所以要使用dump传递到文件中，那么文件的打开方式应该要用 “wb” 的方式打开，那么load呢，读取文件的方式也应该是 “rb” 的方式读取
  • pickle模块序列化后，在反序化列出的结果，跟原先的一模一样
pickle模块 说明
pickle.dumps() 序列化
pickle.dump() open()中使用的，将序列化后的数据传递到文件中
pickle.loads() 反序列化
pickle.load() open()中使用的，将文件的数据反序列化

shelve 模块

shelve 比较特殊，他就1个方法，打开或者创建一个文件。

他就像是把一个文件模拟成字典一样。存储就是一个 key对应一个value，读取也是拿key读取value

shelve 模块 说明
fb = shelve.open() 打开或者创建一个shelve使用的文件
fb.close 关闭也是使用 .close
