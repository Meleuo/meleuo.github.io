Socket模块

socket有2种通信方式，一种是基于文件的 AF-UNIX 的socket，一种是基于网络的 AF-INET

AF-UNIX 是网络socket的前身，目前AF-UNIX应用在一台主机上应用程序交换数据的场景，比如php-fpm和nginx，

AF-INET 应用在网络中不同主机的数据交互

socket网络通信又分为tcp和udp，socket模块默认使用tcp协议

TCP ：SOCK_STREAM() 面向数据流，可靠的通信方式

UDP：SOCK_DGRAM 面向数据报文，不可靠的通信方式

Socket 模块方法

Socket 服务端方法 说明
.bind() 为socket绑定IP + 端口，传递一个元组进去 [0]是ip，[1]是要监听的端口
.listen() 开始监听端口
.accept() 开始等待客户端连接，返回值一个元组[0]是本次连接的对象，[1]是客户端的IP +端口
Socket 客户端方法 说明
.connect() 连接服务端 IP+PORT，以一个元组传递进去，服务端无法理解将会报错
.connect_ex() 连接服务端 IP+PORT，以一个元组传递进去，如果服务端无法连接将会返回一个错误码
Socket 模块 公用方法 TCP 说明
.send() 发送数据给对端，数据格式使用bytes
.recv() 接受对端发送过来的数据，收到的数据格式是bytes
.sendall() 尝试把数据一次性全部发出^注释
Socket 模块 公用方法 UDP 说明
.sendto() UDP发送数据方法需要传递2个参数，第一个是要发送的数据，第二个是对端的ip +port 元组形式，发送的数据需要是bytes
.recvfrom() UDP接收数据，接收2个参数，第一个是数据，第二个是对端的 ip+port
Socket 阻塞设置 说明
.setbloking() 设置套接字的阻塞与非阻塞，传递一个bool，False为非阻塞，True为阻塞default
.settimeout() 设置套接字的等待时间，阻塞和非阻塞都适用

粘包是TCP才会有的问题，UDP是不存在的

TCP 粘包

粘包的本质就是 接受端因为不知道要如何接受服务端数据（收到的数据或大或小），造成收到的数据混乱。

为什么会发生粘包？

首先要了解：TCP传输数据 有 合包，拆包的机制，称之为 nagle 算法

  • 合包：服务端发送数据时，如果多次连续发送，且每个数据过小，前一个数据包就会在缓冲区等待后面的，然后使用nagle算法把这几个数据包打包成一个大的数据包，在发送
  • 拆包：客户端接受数据包时，数据包到达缓冲区，在使用nagle算法把这个大的数据包解分为原来的小数据包在按照顺序挨个接受。[^mtu]

[^mtu]: 数据的拆包还受数据链路层（物理网卡）最大传输单元 MTU的影响，通常机器的 MTU都是1500，也就是说大于1500的数据包在这里还会进行一次拆包，到了应用层（操作系统） 在进行合包

图解TCP粘包

由于服务端并不知道客户端发送发送的数据包字节大小，因此，服务端在接受数据时会出现这4种情况

  • D1和D2 2个数据包正常分开发送，服务器正常接受
  • D1和D2 被nagle 合包成一个数据包，发送给服务端，但是因为偶然，数据包没有进行正常拆包，全部被第一个 recv(1024) 接受到，而第二个没有接收到数据，这样就引发了数据紊乱，也就是粘包
  • D1 D2 D3正常分开发送，D1正常接收，但是第二个数据包 4096 的大小超过服务端准备接受的大小，于是服务端只收到了D2 数据中 1024个字节的数据，剩余的3072被 第三个 recv(1024) 拿过去了一点~这样就有造成了数据紊乱

UDP收发限制

TCP协议如果单次发送的数据如果过大，就会被nagle算法拆包成多个合乎大小的数据包发送，到对端后在缓冲区按照顺序在进行合包^注释，同时也是因为TCP协议发送数据是使用数据流形式，所以是不能控制哪个数据包先到。这也是引发粘包的问题之一

UDP协议的数据传输格式是数据报格式，每次发送数据最大只能发送 65535-20(IP包头)-8(UDP包头)=65507，所以UDP就像是一架到美国的飞机， 一次只能发送 65507， 多了就无法起飞(报错)，一旦发送事故就是全部凉凉。

而且在经过数据链路层（物理网卡）时，由于单次最大传输单元MTU的1500限制，大于1500的UDP数据包，也必须在这里进行拆包，但是由于UDP协议的数据本身就是一次性发送，不可靠的协议，在这里进行拆包过程中有一个数据包丢了，本次传输就失败了~所以使用UDP传输数据要避免拆包，尽可能的保证数据成功发送

单次UDP数据包发送的最佳大小就是：1500-20(IP包头)-8(UDP包头)=1472
