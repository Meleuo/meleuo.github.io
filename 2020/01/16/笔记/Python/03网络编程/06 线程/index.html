线程

如何区分进程和线程？

想象公司的一个部门，有很多人组成，每个人就是一个线程，然后一起工作，每个人都是独立的个体，他们可以沟通，但是需要张嘴说出来（Value,Manager），想添加一个进程就需要新招人，需要发放工资（CPU资源），依次在新招人的成本高一点，招到人开始工作这个流程也就费点时间。

想象公司的一个部门，只有一个人，但是这个人会分身术，分身一起工作，分身是由本体控制的，因此本体想什么分身就能明白（主进程可以和线程共享内存），但是分身和分身之间是感知不到对方的，并且由于分身就由本体幻化一个出来，体质特征和本体无差（PID），并且分身消耗的也是本体的精力，创建一个分身对于本体就是一个念想的事就很快。但是分身存在的前提是必须得有一个本体（进程）。

进程由 代码段  数据段  PCB组成（process control block）
线程由 代码段  数据段  TCB组成（thread control block）

什么是线程

线程被称之为轻量级的进程，

  • 线程是计算机的基本执行单位，最小可执行的就是线程
  • 进程是计算机资源分配的基本单位，线程是可悲执行的基本单位，也是可以被调度的基本单位
  • 线程无法拥有自己的独立资源，线程的执行，必须依赖与所属进程中的资源。
  • 每个进程中至少会有一个线程。
  • 线程又被分为用户级线程和内核级别线程
    • 用户级别线程：对于程序员来说，这样的线程是完全可以被程序员所控制执行调度的。

threading.Thread线程的使用

进程怎么玩，线程就怎么玩

线程和进程一样默认是异步执行。使用 join()同步

线程和进程对比区别

在同一个进程内，所有线程共享这个进程的pid，也就是说所有线程共享所属进程的所有资源和内存地址

线程共享所属进程的所以资源和内存地址，多线程之间互相不能共享。多进程之间互相完全独立，想要共享必须使用 Value或者Manager等方法

线程和进程对比效率

    • 对于计算机内核来所的，这样的线程完全可以被内核控制调度

GIL全局解释锁

只要Cpython解释器才有的

锁的是线程，是CPython解释器上的一个锁，锁的是线程，意思是在同一时间只允许一个线程访问cpu

对于线程来说因为有了GIL，所以线程是没有真正意义上的并行的。

GIL全局解释锁的作用是

  • 同一时间只允许有一个线程使用 资源（例如变量）
  • 限制每个线程的执行事件，一般是 5ms左右
  • 限制线程执行固定的byte code
