协程

操作系统中最小的资源分配单位是进程,线程是CPU调度的最小单位.

线程的初衷就是为了在一个进程内实现并发

但是无论是进程还是线程都需要CPU去调度,其创建,管理

随着我们对效率的要求越来越高, 创建线程所消耗的资源和时间.也开始无法忍受.

这时一个能够在线程内并发的需求诞生了,为之出现的就是: 协程

回顾一下什么是并发: CPU 在同一个时间段类轮流执行多个任务, 多个任务之间的切换受GIl锁, 和时间片的影响. 里还有一步就是切换到下一个任务时 保存状态 这个动作, 以便于接下来能够继续执行, 并发的本质就是在多个任务之间的: 切换+保存状态

协程: 单线程下的并发,又称之为微线程, 纤程,

  • 线程的并发就是由CPU和CIL锁控制的, 这种情况被称之为内核级别的调度. 这样的并发是没有规律的,就是为了并发而调度.(单线程遇到IO或者执行时间过长也会被迫交出CPU权限,切换到其他程序)

  • 纤程的调度是由用户设置的,这种情况被称为应用级别的调度,单线程内的纤程一旦遇到IO操作就会立马切换到其他程序,由此来提升效率.

yield 保存状态

协程底层保存状态调用的也是yield, yield也是python中唯一一个可以保存程序运行状态的方法

从效率对比可见,单纯的保存状态执行是不能提高效率, 反而还没有串行快,这是因为上面yiled中有一个while,

对于单线程下,不可避免的出现IO操作, 协程就是控制单线程下的多个任务有一个任务遇到了IO时立马切换到另外一个任务去计算.因为切换的操作属于用户在程序内控制协程的切换, 这样程序级别的切换,不需要操作系统调度, 因此非常的轻量级.

这样就在单线程中实现了并发的效果,且最大限度的利用了CPU的资源

Gevent 协程

gevent 说明
func = gevent.spawn(function,参数) 把function注册一下,返回一个对象func
gevent.join(func) 等待func指向的函数function执行完毕,在此函数执行遇到IO后会自动切换
gevent.joinall([f1,f2,f3]) 传递一个列表,等待列表中对象对于的函数执行完毕 ,在此函数执行遇到IO后会自动切换
gevent.sleep(num) gevent 的一个主动IO阻塞的方法

gevent遇到IO后自动切换

默认情况下Gevent是只认识自己内置的 IO操作的,其他的IO他都不认识,也就无法正常的切换程序
