Docker 镜像删除

Docker 的删除应该是先删除 由该镜像创建的容器，再删除镜像，不然会报错，可以使用 -f 强制删除，但是不建议这么做

当一个镜像有多个标签时，首先删除的不过是对应的标签，使用 docker images ，当一个镜像只存在一个标签时，删除操作会彻底删除该镜像

Docker Hub

Docker Hub 是docker官方维护的docker公有仓库，docker search 默认操作的也是 Docker hub

自动构建

如果业务中某些镜像中的配置需要经常变化，使用自动构建就会非常方便

自动构建就是 Docker Hub 制动跟踪一个 Git网站（目前支持 Githu和 BitBucket）上的项目，一旦发现新的提交，触发自动构建

具体流程如下

  1. 创建登录Dcoker Hub 以及目标网站
  2. 再目标网站中链接Docker Hub 的账户
  3. 再 Docker HUb 上配置一个自动构建
  4. 选取目标网站上项目的一个项目，需要包含 Dockerfile 的分支
  5. 指定 Dockerfile 的位置 并提交构建

请不要挂载指定文件到docker 容器中。 因为，在宿主机中使用文本编辑工具(vim, sed 等) 都会改变文件的inode号，从docker 1.1.0开始，这会引起报错

Dockerfile

一个基基础础的Dockerfile 配置有4个部分，分别是

配置参数

FROM

指定构建的基础镜像

MAINTAINER

指定维护者的信息

RUN

或者使用 RUN [‘executable’, ‘param1’, ‘param2’]

前者是在shell中命令， 即在 /bin/sh -c 上执行的。

后者则是使用 exec执行， 指定使用其他终端或者通过第二种方式实现，例如 RUN [“/bin/bash” , “-c” , “echo Hello”]

每条RUN指令都将在当前镜像基础上执行命令，并提交为新的镜像， 可以使用 \ 换行

CMD

支持3种格式

指定启动容器时执行的命令，每个Dockerfile只能有一个CMD命令， 如果有多条，只有最后一条会被运行

如果用户启动时指定了运行命令，则会覆盖掉CMD

EXPOSE

告诉Docker 宿主机 容器内服务需要暴露的端口号

例如： EXPOSE 80 443

ENV <key> <value>
设置一个环境变量, 可以供后续的 RUN命令使用

ADD <src> <dest>

复制 到容器的 中， 其中 src 是dockerfile文件所在的相对路径，可以是文件或者目录，目标dest 如果不存在则会新建

  • 可以是一个 url路径
  • 可以是一个tar文件， 会自动解压到 dest 中

COPY <src> <dest>

复制 到容器的 中， 其中 src 是dockerfile文件所在的相对路径，可以是文件或者目录，目标dest 如果不存在则会新建

ADD 和COPY 之间，如果是本地的文件推荐使用 COPY

ENTRYPOINT

VOLUME

创建一个可从本地主机或其他容器的挂载点

USER dameon

指定容器运行时使用的用户名 或者UID， 当服务不需要使用 root时可以使用这个

  • RUN groupadd -r nginx && useradd -r -g nginx nginx
  • 临时获取管理权限推荐使用 gosu 而不是 sudo

WORKDIR <path>

为后续的 RUN, ENTRYPOINT, CMD 指定运行目录

可以使用多个 WORKDIR，后续命令使用的如果是相对路径则基于 WORKDIR定义的目录
